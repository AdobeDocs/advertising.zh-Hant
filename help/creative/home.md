# 2.0 版本的新內容[!DNL Creative]

以下功能為新增或近期變更。

| 日期 | 特色 | 說明 | 欲了解更多資訊 |
| ---- | ------- | ----------- | -------------------- |
| 2026年1月21日 | 創意圖書館，Adobe GenStudio 用於績效行銷 | 使用 [Adobe GenStudio for Performance Marketing](https://experienceleague.adobe.com/en/docs/genstudio-for-performance-marketing/user-guide/home) 的廣告主現在可以將 GenStudio 的展示廣告體驗匯入您的創意資料庫。 <!-- Each GenStudio experience is saved as an HTML5 creative.--> 你可以將引進的創意納入你的廣告創意體驗中。 | 請參見「[關於您的創意圖書館](/help/creative/creative-libraries/creative-libraries-about.md)」及「[將標準創意作品加入創意圖書館](/help/creative/creative-libraries/creative-add-standard.md)」。 |
|  | 標準創意 | 現在，建立圖片和 HTML5 廣告的選單選項合併成一個「[!UICONTROL Standard Display]」選項，你也可以用它從 GenStudio 匯入展示廣告體驗。 | 請參見「[關於您的創意圖書館](/help/creative/creative-libraries/creative-libraries-about.md)」及「[將標準創意作品加入創意圖書館](/help/creative/creative-libraries/creative-add-standard.md)」。 |
| 2025年12月17日 | 警報 | [!UICONTROL Pulse Panel]檢視中的[!UICONTROL Experiences]圖示顯示是否有任何提醒可供你的體驗。你可以打開面板查看每個警示，並選擇性地開啟相關的經驗或創意，以便必要時編輯以解決問題。 你也可以選擇性地關閉任何警報。<br><br>對於擁有廣告 DSP 的廣告主，關於由 [!DNL Creative] 體驗建立的 DSP 投放廣告的警示，會在 DSP 的活動層級提供。 | 詳見「[查看提醒」。](/help/creative/reports/alerts-view.md) |
| 2025年11月26日 | 廣告創意的成本報告 | （僅限廣告創意與廣告DSP的廣告主;測試版功能）新 [!UICONTROL All-in Cost] 報告已結束測試版，並普遍可查閱。 | 請參見「[關於自訂報告](/help/creative/reports/reports-about.md)」和「[管理自訂報告](/help/creative/reports/report-manage.md)」。 |
| 2025年11月17日 | 廣告創意的成本報告 | （僅限廣告創意與廣告DSP的廣告主;測試版功能）新 [!UICONTROL All-in Cost BETA] 報告顯示 Adobe Creative 廣告服務的支出有多少。 報告的支出指標包括[!UICONTROL Billable Creative Spend]、、[!UICONTROL Non Billable Creative Spend]、 [!UICONTROL Total Creative Spend]&#x200B;[!UICONTROL Total Creative CPM]&#x200B;[!UICONTROL Estimated Tax on Creative]及 。你可以在活動、包裝、置入和廣告層級查看創意、屬性、目標及其他資料。 | 請參見「[關於自訂報告](/help/creative/reports/reports-about.md)」和「[管理自訂報告](/help/creative/reports/report-manage.md)」。 |
| 2025年9月24日 | 充滿活力的創意人 | 現在有一種更簡單的自助式工作流程，可以利用廣告變體目錄中的資料產生動態廣告，這些資料會經過廣告範本。 | 請參見「[動態廣告](/help/creative/introduction/workflow-dynamic-ads.md)工作流程」。 |
| 2025年9月3日 | 創意優化與排程 [!DNL experiences] | 用來設定創意優化與排程的選單選項或按鈕現在稱為「[!UICONTROL Creative Optimization]」，而非[!UICONTROL Edit Schedules]「」或「[!UICONTROL Add Schedules]。」 | 參見「[為目標體驗](/help/creative/experiences/experience-optimization-scheduling-targeting.md)自訂創意優化與排程」及「[為無目標](/help/creative/experiences/experience-optimization-scheduling-no-targeting.md)設定的體驗自訂創意優化與排程」。 |
|  | 廣告輪替與排班 [!DNL experiences] | 一種新的創意輪替類型「，[!UICONTROL Sequencing]」會依特定順序旋轉相關的創意組合包（先服務組合包 1，組合包 2 次，依此類推），並設定每個組合序列中指定的總曝光次數。 你可以設定序列中的最後一個組合，a） 無限顯示，或 b\） 迴回第一個組合包。 | 參見「[為目標體驗](/help/creative/experiences/experience-optimization-scheduling-targeting.md)自訂創意優化與排程」及「[為無目標](/help/creative/experiences/experience-optimization-scheduling-no-targeting.md)設定的體驗自訂創意優化與排程」。 |
| 2025年8月4日 | 廣告的目標受眾 [!DNL experiences] | 更複雜的受眾納入與排除邏輯——廣告DSP中同樣的邏輯——在受眾目標設定中也可用。 當你為廣告體驗設定受眾目標時，現在可以設定多個分區群組，並根據邏輯指定是否包含或排除使用者。<br><br>您可以查看每個分段的總觀眾人數，以及每個分段群組的活躍與總觀眾人數。<br><br>有了這個選項，原本的「」[!UICONTROL Split targets to create nodes]選項就不再可用了。<br><br>如果你已有針對多個細分群的受眾節點經驗，現在透過「[!UICONTROL Include Any]」選項來鎖定目標，且使用者必須只屬於其中一個細分群才能獲得曝光。 過去，使用者必須屬於所有區段。 | 參見「[在最終層級](/help/creative/experiences/experience-target-node-add-final.md)新增目標節點」、「[在節點](/help/creative/experiences/experience-target-node-add-inner.md)間插入目標節點」以及「[新增兄弟目標節點](/help/creative/experiences/experience-target-node-add-sibling.md)」。 |
| 2025年7月10日 | 影片創作者 | 目前支援第一方影片創意以及針對影片的組合與體驗：<ul><li>你現在可以上傳第一方影片創意，並加入專屬影片的組合包中。 在組合設定中，「[!UICONTROL Bundle Type]」選項現在包含 [!UICONTROL Standard Display]、 [!UICONTROL Dynamic Display]和 [!UICONTROL Standard Video]。</li><li>你可以用影片組合來創造專屬影片的廣告體驗。 廣告體驗設定現在包含[!UICONTROL Ad Type]一個帶有 [!UICONTROL Standard Display]、 [!UICONTROL Dynamic Display]、 和 [!UICONTROL Video] 選項的「」設定。 你可以根據點擊率、完成率或自訂目標來優化影片廣告。</li><li>影片廣告體驗標籤是依影片時長和位元率定義，而非廣告大小。</li><li>影片廣告會自動轉碼成 Adobe Advertising DSP 編碼，方便你預覽。 你可以選擇性地將其他 DSP 的轉碼套用到內 [!UICONTROL Tag Manager]的任何廣告經驗標籤。</li></ul> | 請參閱「[關於您的創意庫](/help/creative/creative-libraries/creative-libraries-about.md)」、「[管理創意套件](/help/creative/creative-libraries/bundle-manage.md)」、「[目標體驗設定](/help/creative/experiences/experience-settings-targeting.md)」、「[非目標體驗設定](/help/creative/experiences/experience-settings-no-targeting.md)」以及「[自訂影片廣告體驗標籤](/help/creative/experiences/experience-tag-video-transcoding.md)的轉碼選項」。 |
| 2025年5月21日 | [!UICONTROL Creative Libraries] | 你現在可以將 Adobe Experience Manager 資產庫中的圖片加入你的 [!UICONTROL Creative Libraries] 廣告體驗中，這樣就能在廣告體驗中使用。 | 參見「[配置 Adobe Experience Manager 影像資產](/help/creative/creative-libraries/aem-assets-configure.md)的存取」及「[將標準創意作品加入創意庫](/help/creative/creative-libraries/creative-add-standard.md)」。 |
| 2025年2月10日 | [!UICONTROL Creative Libraries] | 之前你只有一個創意圖書館。 現在，你可以為每個廣告主建立多個函式庫。 | 請參見「[關於你的創意圖書館](/help/creative/creative-libraries/creative-libraries-about.md)」。 |
| | [!UICONTROL Creative Libraries] > [!UICONTROL Creatives] | [!UICONTROL Creatives]視圖包含 和 [!UICONTROL Standard Ads]的分頁[!UICONTROL Dynamic Ads]。<ul><li>這個 **[!UICONTROL Standard Ads]分頁** 允許你上傳和管理圖片、HTML5、彈性HTML5以及第三方創意。</li><li>這個 **[!UICONTROL Dynamic Ads]** 分頁允許你管理由上傳的動態檔案使用定義廣告範本產生的動態廣告;過去動態廣告是在 [!DNL Adobe Advertising Dynamic Creative Optimization (DCO)]。<br><br> 目前，你可以預覽、複製並刪除動態廣告。 你也可以將動態廣告附加到創意組合包中，以達成目標廣告體驗，或在非目標體驗的廣告標籤上附加。 只有管理員使用者能動態產生廣告。</li></ul> | 請參見「[關於你的創意圖書館](/help/creative/creative-libraries/creative-libraries-about.md)」。 |
| | [!UICONTROL Creative Libraries] > [!UICONTROL Bundles] | 將多位創意作品合併成一個 *組合* ，方便他們加入一個體驗中。 你可以建立標準廣告包，並在上面附加標準創意。 同樣地，你也可以建立動態廣告組合，並將動態創意附加到上面。 | 請參閱「[管理創意套件](/help/creative/creative-libraries/bundle-manage.md)」。 |
| | [!UICONTROL Experiences] | 在新的廣告體驗設定中，你現在可以指定體驗是否使用決策樹目標，且儲存後無法更改設定。 有決策樹目標設定的經驗與沒有決策樹目標的經驗工作流程是不同的。 | 參見「[以目標設定](/help/creative/experiences/experience-create-targeting.md)創造體驗」和「[不針對性的](/help/creative/experiences/experience-create-no-targeting.md)體驗體驗」。 |
| | [!UICONTROL Experiences] | 你現在只能用單一創意庫的創意組合包來創造目標體驗，而非單一創意。 你仍然可以將單一圖書館的個別創意連結到非目標體驗，無需決策樹鎖定。<br><br>由於結構性變動，您的舊有體驗將在今年晚些時候被淘汰。 | 自助顧客：在新的使用者介面中重建您的體驗。 參見「[用目標建立](/help/creative/experiences/experience-create-targeting.md)體驗」。<br><br>管理服務客戶：您的 Adobe 帳戶團隊將在新的使用者介面中重建您的體驗。 |
| | [!UICONTROL Experiences] | 擁有廣告 DSP 的廣告主可選擇直接將標籤上傳至 Advertising DSP 活動中作為廣告。 | 請參見「[匯出並實作即時體驗](/help/creative/experiences/experience-tag-export.md)的廣告體驗標籤」 |
| | DCO 經歷 | 舊有的 DCO 體驗將在今年稍晚被淘汰。 你的 Adobe 帳戶團隊會將你的 DCO 體驗重建為動態廣告 [!DNL Creative]，但帶有額外目標鎖定的 DCO 體驗會被重新建立為 [!DNL Creative] 體驗。 | — |
| | 廣告標籤 | 廣告伺服器端點會切換為廣告 DSP 廣告伺服器。<br><br>新的廣告標籤包含額外參數，用於傳遞通用 ID（除了 Cookie ID）。 | 自助顧客：一旦體驗在 [!DNL Creative]，請替換現有活動中的廣告標籤。 請參閱「[匯出並實作廣告體驗標籤以取得即時體驗](/help/creative/experiences/experience-tag-export.md)」。<br><br>託管服務客戶：您的 Adobe 帳戶團隊將取代現有廣告活動中的廣告標籤。 |
