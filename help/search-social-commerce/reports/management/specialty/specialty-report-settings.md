|  | [!UICONTROL Data Aggregation] | ([!UICONTROL Forecast Accuracy Report only])應顯示在報告每一列的時間單位： <i>[!UICONTROL Summary (the default)]</i>、<i>[!UICONTROL Daily]</i>、<i>[!UICONTROL Weekly]</i> （然後指定一週的第一天）、<i>[!UICONTROL Monthly]</i>、<i>[!UICONTROL Quarterly]</i>、<i>[!UICONTROL Yearly]</i>或<i>[!UICONTROL Day of Week]</i> （僅基本報告）。<br><br>報告會針對指定日期範圍內的每個時間單位，為每個關鍵字或其他專案顯示一列。 例如，如果您為關鍵字報告選取<i>[!UICONTROL Summary]</i>且日期範圍為<i>[!UICONTROL Last 7 Days]</i>，則報告會針對每個關鍵字包含一個資料列，每個資料列都會彙總七天內的總計和平均值。 具有<i>[!UICONTROL Daily]</i>資料彙總的相同報表在過去七天中的每一天，每個關鍵字都包含一列（每個關鍵字包含七列）。<br><br><b>附註：</b><ul><li>當您使用&quot;[!UICONTROL Hourly]&quot;或&quot;[!UICONTROL Day of Week (Hourly)]&quot;彙總每小時彙總資料時：</li><ul><li>您最多可以檢視過去14天的資料。 報表會以24小時制依小時顯示資料。 「[!UICONTROL 0]」在廣告商的時區中包含00:01-00:59 (12:01-12:59 a.m.)和「[!UICONTROL 23]」包含23:01-23:59 (11:01-11:59 p.m.)。</li><li>不支援&#39;&#39;[!UICONTROL Device]&#39;&#39;欄。</li><li>不支援[!UICONTROL Ad Group Report]的[!UICONTROL Data Based On]選項[!UICONTROL Portfolio Configuration During the Specified Dates]。</li><li>對於[!UICONTROL Ad Group Report]，您只能以目前的產品組合組態（使用&quot;[!UICONTROL Data Based On]&quot;選項&quot;[!UICONTROL Current Portfolio Configuration]&quot;）作為每小時資料的基礎，而不是以指定日期的產品組合組態為基礎。</li></ul></ul> |
|  | [!UICONTROL Conversions Based on] | （僅限[!UICONTROL AdWords Shopping Performance Report]、[!UICONTROL Campaign Daily Impression Share Report]和[!UICONTROL Keyword Daily Impression Share Report]）如何報告轉換資料：<ul><li><i>[!UICONTROL Transaction date]</i> （預設值）：檢視交易日期在指定時段內發生的交易。 此選項會顯示指定期間內已賺取的收入。</li><li><i>[!UICONTROL Click date]：</i>檢視在指定時段內發生的點按所產生的交易。 當產品組合在點按和交易之間有重大延遲時，此選項可用於計算產品組合的每次點按歷史收入，其指示一段時間內預期的收入行為。</li></ul> |
|  | [!UICONTROL Date Range] | 要產生資料的日期範圍：<ul><li><i>[預設範圍]：</i>常用時間增量清單，範圍從<i>[!UICONTROL Today]</i>到<i>[!UICONTROL Last 180 Days]</i>。 預設值為<i>[!UICONTROL Last 7 Days]</i>，其會報告資料可用的過去七天。 <b>附註：</b> <i>[!UICONTROL Last Month]</i>、<i>[!UICONTROL Last 3 Months]</i>和<i>[!UICONTROL Last 6 Months]</i>會顯示前一個日曆月的資料。</li><li><i>[!UICONTROL Custom Date Range]：</i>指定開始日期和結束日期。 以MM/DD/YYYY或M/D/YYYY格式輸入日期，或按一下欄位旁的![行事曆](/help/search-social-commerce/assets/calendar.png "行事曆")並選取日期。</li></ul> |
|  | [!UICONTROL Compare With] | 比較指定日期範圍的資料與第二日期範圍的資料。 選取此選項時，會為每個一般資料欄新增兩個額外的欄。 例如，報告不只包括「曝光數」的一欄，而包括「[!UICONTROL Impressions Range 1]」、「[!UICONTROL Impressions Range 2]」和「[!UICONTROL Impressions Difference]」的欄。<br><br><b>附註：</b><ul><li>自訂/衍生量度的差異欄不會顯示。</li><li>比較大型日期範圍的報表需要較長時間才能產生。</li></ul> |
|  | [!UICONTROL Show Comparison Data as] | （選取&quot;[!UICONTROL Compare With]&quot;時）如何在&quot;[資料欄位] [!UICONTROL Difference]&quot;欄中表示兩個所選日期範圍之間的資料差異：<ul><li><i>[!UICONTROL Variance]</i> （預設）：將差異顯示為數值。</li><li><i>[!UICONTROL % Change]：</i>以百分比顯示差異。</li></ul> |
|  | [!UICONTROL Filter By] | （僅限[!UICONTROL RSA Asset Report]、[!UICONTROL Campaign Daily Impression Share Report]和[!UICONTROL Keyword Daily Impression Share Report]）是否針對特定產品組合或特定廣告網路的資料進行報告：<ul><li><i>[!UICONTROL Portfolio]</i> （預設）：若要在一或多個產品組合中包含行銷活動的資料。</li><li><i>[!UICONTROL Search Engine]：</i>若要將行銷活動的資料包含在一或多個廣告網路中（如適用於報表型別）。 |
|  | \[主要篩選器\] | 要包含的資料元件。 如果您未進行選取，則所有產品組合（如適用）或所有適用廣告網路及其子元件的資料都會納入報告中。 您可選擇指定個別元件和子元件，以縮小要報告的資料範圍。 根據報表型別以及（若適用）您是依產品組合或廣告網路進行篩選，請指定要包含的元件：<ul><li><i>[!UICONTROL Portfolio]：</i>一或多個產品組合或其子元件（行銷活動或廣告群組）。 若要選取元件及其所有子元件，請選取元件名稱旁的核取方塊。 若要選取子元件，請選取子元件名稱旁的核取方塊，然後按一下>>將其移動到[!UICONTROL Selected Filters]欄。 例如，若要取得Portfolio1及其所有促銷活動和廣告群組的資料，請選取Portfolio1旁的核取方塊。 若要在Portfolio1的Campaign 1中發生至少一個事件時取得資料，請展開Portfolio1，然後只選取Campaign 1旁的核取方塊。</li><li><i>[!UICONTROL Search Engine]：</i>一或多個廣告網路或其子元件（帳戶、行銷活動或廣告群組）。 若要選取元件及其所有子元件，請選取元件名稱旁的核取方塊，然後按一下>>將其移動到[!UICONTROL Selected Filters]欄。 若要選取子元件，請選取子元件名稱旁的核取方塊，然後按一下>>將其移動到[!UICONTROL Selected Filters]欄。 例如，若要在任何[!DNL Google Ads]帳戶、行銷活動和廣告群組中發生至少一個事件時取得資料，請選取[!DNL Google AdWords]旁的核取方塊。 若要只取得[!DNL Google Ads]帳戶1中促銷活動1的資料，請展開[!DNL Google Ads]，然後展開帳戶1，然後只選取促銷活動1旁的核取方塊。</li></ul><b>附註：</b><ul><li>若要展開清單中的元件（例如列出廣告網路上的帳戶），請按一下元件旁的![向右箭號圖示](/help/search-social-commerce/assets/compressed-item.png "向右箭號圖示")。</li><li>若要檢視任何專案的元件型別，請將游標停留在它上。</li><li>依預設，僅會列出a)作用中及最佳化的產品組合及其作用中元件，或b)作用中及啟用的廣告網路帳戶、行銷活動及其作用中元件。 若要檢視暫停和刪除的元件，請按一下&#x200B;**[!UICONTROL Show]**&#x200B;旁的![向下箭頭](/help/search-social-commerce/assets/arrow-down-expand.png "向下箭頭")，然後選取&#x200B;**[!UICONTROL All]**。</li><li>當您依產品組合產生進階報表時，產生的資料會用於目前對應至指定產品組合的行銷活動。 此報表不包含日期範圍內位於產品組合中，但仍不存在的行銷活動的資料。</li></ul> |

|  | [!UICONTROL Order Results/Limit Rows by] | 最多可依報表中包含的兩個欄來排序報表。 每種報表型別的預設值都不同。 若要自訂排序順序，請選取報表欄，然後選取<i>[!UICONTROL Ascending]</i> （顯示從A到Z或從1到100的結果）或<i>[!UICONTROL Descending]</i> （顯示從Z到A或從100到1的結果）。 指定至少一個要排序的資料行。 如果您依兩欄排序，則報表會先依指定的第一欄排序，然後再依指定的第二欄排序。 |
|  | [!UICONTROL Include assets with no performance data, except for assets with zero lifetime data] | ([!UICONTROL RSA Asset Report only])包含指定日期沒有可用效能資料的列，為遺失的資料插入零(0)值。 依預設，此選項未選取，只有在資料（無論值為何）可用時才會顯示列。<br><br>選取此選項時，報表會包含不含行銷活動的廣告網路帳戶、不含有效關鍵字的行銷活動以及整個資料範圍內已停用、暫停和刪除之元件的效能資料。 此外，RSA資產報告會顯示所有資產的列，但期限資料為零的資產除外。<br><br><b>警告：</b>如果您選取此選項，並為許多沒有資料的子元件建立大型日期範圍的報告，則報告可能需要很長時間才能產生。 |
|  | [!UICONTROL Share with others] | 可讓有權存取相同廣告商資料的其他使用者檢視產生的報告，以及（如果您將報告另存為範本）使用範本但不編輯或刪除它。 依預設，不會選取此選項。 <b>注意：</b>無論此設定為何，您的報表和範本一律對更高層（管理員）角色的所有使用者以及任何指派的Adobe帳戶團隊成員可見。 |
| [!UICONTROL Advanced Filters] | \[進階篩選器\] | 只有在量度的值符合指定條件時，才會傳回列；量度不需要納入報表當作欄。 可用量度的清單會依報告型別而異，但可能包括廣告商的自訂衍生量度、每個廣告網路和產品組合元件（例如[!UICONTROL Campaign ID]和[!UICONTROL Campaign Status]）的ID和屬性名稱、廣告商的轉換量度，以及廣告網路中與點選相關的量度。 可用的運運算元包括<i>[!UICONTROL contains]</i>、<i>[!UICONTROL starts with]</i>、<i>[!UICONTROL equals]</i>、<i>[!UICONTROL is greater than]</i>、<i>[!UICONTROL is greater than or equal to]</i>、<i>[!UICONTROL is less than]</i>、<i>[!UICONTROL is less than or equal to]</i>或<i>[!UICONTROL isn't equal to]</i>。<br><br>若要套用一或多個篩選器，請執行下列動作：<ul><li>選取量度和運運算元，然後輸入適用的值。 例如，若要只傳回點選超過100次的關鍵字，請選取[!UICONTROL Clicks]，再選取[!UICONTROL >]，然後在輸入欄位中輸入100。</li><li>（若要套用其他篩選器）對於每個其他篩選器，按一下&#x200B;**[!UICONTROL +Add Filter]**，選取&#x200B;**[!UICONTROL AND]**&#x200B;或&#x200B;**[!UICONTROL OR]**，選取量度和運運算元，然後輸入適用的值。</li></ul> |
| [!UICONTROL Attribution] | [!UICONTROL Rule] | (僅限具有Adobe Advertising畫素式轉換追蹤服務的廣告商；[!UICONTROL AdWords Shopping Performance Report]、[!UICONTROL Campaign Daily Impression Share Report]及[!UICONTROL Keyword Daily Impression Share Report]))在報表中，如何歸因轉換資料mdash；可能跨多個廣告管道及產品組合mdash；在一連串導致轉換的事件中：<ul><li><i>[!UICONTROL First Event]：</i>將轉換歸因於廣告商的[點按回顧期間](/help/search-social-commerce/glossary.md#c-d)內系列中的第一個付費點按，如果沒有發生付費點按，則歸因於廣告商的[點按回顧期間](/help/search-social-commerce/glossary.md#i-j)內的最後一次點按。</li><li><i>[!UICONTROL Weight First Event More]：</i>將轉換歸因於發生在廣告商的[點按回顧期間](/help/search-social-commerce/glossary.md#c-d)和[曝光回顧期間](/help/search-social-commerce/glossary.md#i-j)內的系列中的所有事件，但是會給予第一個事件最大的權重，並連續減少對以下事件的權重。 當轉換前同時有付費點按和曝光時，指定的曝光覆寫權數會進一步套用至曝光。 當轉換之前只有曝光次數時，曝光次數會根據廣告商的瀏覽權數（而非曝光次數覆寫權數）加權。</li><li><i>[!UICONTROL Even Distribution]：</i></li> 將轉換平等地歸因於系列中的每個事件廣告商的瀏覽權數，而不是印象覆寫權數。</li><li><i>[!UICONTROL Weight
