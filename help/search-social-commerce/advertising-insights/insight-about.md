# 關於 [!UICONTROL Advertising Insights]

[!UICONTROL Advertising Insights] 針對包含作用中行銷活動的最佳化和作用中產品組合，提供可操作的視覺資料。

每個分析都會隨選產生，而輸出是可以下載的檔案或熒幕報告。 您可以刪除分析檔案的已產生例項。

## 型別 [!UICONTROL Advertising Insights]

| Insight | 說明 |
| --- | --- |
| [!UICONTROL AMO-AA Tracking Discrepancy] | (僅具有Adobe Advertising-Adobe Analytics整合的廣告商)會識別追蹤問題，其特徵為天數的點按執行個體比率超出正常變數，或變數超過20%。 報表包含：<ul><li>A [!DNL Microsoft PowerPoint] 包含資料差異摘要的檔案；顯示具有最多追蹤問題之行銷活動的各行銷活動差異的表格和趨勢圖表；疑難排解建議；以及報告方法的說明。</li><li>A [!DNL Microsoft Excel] 包含產品組合中每個行銷活動的摘要資料，以及每個行銷活動過去月份的每日資料的檔案。</li></ul> |
| [!UICONTROL Attribution Analysis] | ([!DNL PowerPoint] 表示格式)表示不同的歸因模型何時可以改善單一產品組合的收入模型和最佳化。 |
| [!UICONTROL Campaign Caps] | ([!DNL PowerPoint] 簡報格式)表示單一投資組合在過去30天的支出是否受到行銷活動預算上限的限制，並建議調整投資組合設定以實現最佳投資報酬率。 |
| [!UICONTROL Day of Week] | ([!DNL PowerPoint] 簡報格式)表示單一投資組合在過去30天中，依星期幾(DOW)的績效，並建議DOW支出目標以增加您的投資報酬率。<br><br>**注意：** 沒有足夠花費或過去兩天無法花費到目標的Portfolio，無法供深入分析使用。 |
| [!UICONTROL Delayed Revenue] | 測量產品組合的轉換延遲（從廣告點選到後續轉換的經過時間），並顯示加權收入(現在稱為「[目標值](/help/search-social-commerce/glossary.md#o-p)「)、ROI和模型準確性（由於延遲）。 此分析可讓您偵測點按日期歸因的績效量度中最近的變更，並透過點按日期與交易日期來檢視報表的影響。<br><br>分析內容包括 [!DNL Excel] 活頁簿中包含加權收入（現在稱為「目標值」）、收入正確性及延遲因子的個別工作表。 其中也包含 [!DNL Powerpoint] 包含各種圖表的檔案。 |
| [!UICONTROL Event Path] | 識別不同的管道和行銷活動如何透過分析事件路徑和多點觸控轉換來推動轉換。 您可以上傳XLSX和ZIP （壓縮XLSX）格式的檔案，然後a)擷取事件或b)分析分類事件。 |
| [!UICONTROL Google Account Audit] | (適用於 [!DNL Google Ads] 帳戶；對售前最有用)提供帳戶績效的概觀，並著重說明不足之處，以顯示其管理效率。 稽核包括有關競標、比對型別、預算上限、每週績效天數、行動效能等資訊。 若要產生分析，您必須上傳1) [!DNL Google Ads] 來自的帳戶促銷活動、關鍵字及變更歷史記錄報告 [!DNL Google Ads] Web使用者介面和b)帳號的Bulksheet檔案 [!DNL Google Ads Editor] 應用程式。 所有檔案都必須是CSV、TSV、TXT或ZIP （壓縮的CSV、TSV或TXT）格式。<br><br>此深入分析可用於協助潛在客戶瞭解Search、Social和Commerce如何提高效能及解決分析所強調的缺陷。 Adobe客戶團隊也可以使用該見解在正式上線客戶之前評估帳戶。 |
| [!UICONTROL Impression Share Lost] | ([!DNL PowerPoint] presentation format)表示產品組合的預算何時已限制曝光份額 [!DNL Google Ads] 行銷活動，並建議對預算和行銷活動進行變更»[!UICONTROL Multiple]」會據此進行設定。 |
| [!UICONTROL Location Target Performance] | (適用於 [!DNL Google Ads] 行銷活動； [!DNL Excel] 試算表格式)競標調整、點按數、成本和加權收入(現在稱為&quot;[目標值](/help/search-social-commerce/glossary.md#o-p)&quot;)適用於單一產品組合中的每個位置目標。 資料適用於過去90天內的任何日期範圍，且您可以檢視每日結果或每個行銷活動的摘要。 |
| [!UICONTROL Match Type] | 成本和加權收入的熒幕圖表(現在稱為&quot;[目標值](/help/search-social-commerce/glossary.md#o-p)&quot;)在單一產品組合中的相符型別間切換最多90天。 |
| [!UICONTROL Mobile Optimization] | ([!DNL PowerPoint] 簡報格式)依裝置型別顯示過去30天內單一產品組合的效能、評估行動裝置專屬最佳化功能的目前設定，並建議調整設定。 |
| [!UICONTROL Model Accuracy] | 識別搜尋、Social和Commerce預測單一作用中或最佳化產品組合的支出和收入程度，並識別模型不正確的來源。 報表包含：<ul><li>A [!DNL PowerPoint] 包含過去七天之預測與實際成本、點按數、每次點按成本(CPC)、收入、每次點按收入(RPC)及投資報酬率(ROI)摘要的檔案；上個月超支及支出不足競標單位的彙總資料；顯示上個月成本模型準確度、收入模型準確度、預測成本、點按數、CPC、收入、RPC及投資報酬率與實際值的趨勢圖；按點按層級區分的模型準確度；支出不足或超支的最高競標單位；以及改善模型準確度的建議。</li><li>一個 [!DNL Excel] 每個關鍵字的模型精確度與模型精確度的檔案（依日）。</li></ul>此深入分析提供專案組合層級模型準確度報表以外的詳細資訊，這些報表可從 [!UICONTROL Portfolios] 檢視。 |
| [!UICONTROL Normalized Sim (Combined)] | 針對多個投資組合的指定目標支出層級（步驟），傳回最佳支出配置。 此分析使用標準化的加權收入(現在稱為&quot;[目標值](/help/search-social-commerce/glossary.md#o-p)&quot;)根據指定時段內的每週模擬而定。 您可以產生a)每個所選投資組合的個別模擬，或b)具有相同目標和貨幣的多個投資組合的單一模擬。 模擬會以試算表顯示。<br><br>對於單一、多產品組合的模擬，試算表包含兩個工作表：1)趨勢圖，顯示標準化的加權收入(現在稱為「[目標值](/help/search-social-commerce/glossary.md#o-p)「)對於每個目標支出層級（步驟），和2)一個表格顯示每個適用支出層級（步驟）的一列。 為了進行個別產品組合模擬，試算表包含1)摘要工作表，列出所有適用的產品組合和時段，以及2)每個產品組合的兩個工作表： a)趨勢圖表，顯示標準化的加權收入(現在稱為&quot;[目標值](/help/search-social-commerce/glossary.md#o-p)「)針對每個目標支出層級（步驟），以及b)顯示每個適用支出層級（步驟）一列的表格。 |
| [!UICONTROL Portfolio Launch] | ([!DNL PowerPoint] presentation format)識別產品組合是否已準備好啟動（已變更為「最佳化」狀態）。 此報表包含預測績效與實際績效的比較；建議的產品組合開始日期以及應排除在成本與收入預測之外的日期清單；成本、收入、每個成本的收入及隨時間轉換的關鍵字；有關過去兩週支出上限的任何行銷活動的資訊；以及啟動或不啟動的建議。 |
| [!UICONTROL Quality Score] | 顯示具有曝光次數之關鍵字的品質分數趨勢 [!DNL Google Ads] 或 [!DNL Microsoft Advertising] 過去90天內。 它包含a) a [!DNL PowerPoint] 檔案和b) [!DNL Excel] 包含具有最高花費的100個關鍵字和（適用時）品質分數不佳的前100個關鍵字的檔案。<br><br>**附註：**<ul><li>當產品組合有超過20,000個競標單位時，僅納入在過去30天（而非90天）內點選多次的關鍵字。</li><li>含有許多關鍵字資料的深入分析需要較長時間才能產生。</li></ul> |
| [!UICONTROL Query Cross Matching] | ([!DNL Google Ads] accounts)識別 [!DNL Google Ads] 符合多個關鍵字，並提供將流量導向到何處的建議。 使用這些建議來改善關鍵字效能，並因此改善模型準確性。<br><br>此分析包括交叉比對查詢、比對關鍵字和效能資料的試算表；協助您視覺化交叉比對查詢和關鍵字之間關係的網路圖表；基礎搜尋查詢報表；以及如何解譯資料的資訊。 |
| [!UICONTROL Settings Audit] | 指出您的產品組合是否根據最佳實務進行設定。 內容包括：<ul><li>A [!DNL PowerPoint] 此檔案顯示是否根據最佳實務設定您所有產品組合及其元件行銷活動的主要設定、是否有任何行銷活動未指派給產品組合，以及偏離最佳實務的後果。</li><li>一個 [!DNL Excel] 每個重要產品組合設定和重要促銷活動設定都有個別索引標籤的檔案。 資料包含您每個產品組合的產品組合設定，以及不符合最佳實務的每個行銷活動的行銷活動設定。</li></ul> |
| [!UICONTROL Time-of-day Analysis] | （適用於具有的產品組合） [!DNL Google Ads] 搜尋、顯示或購物行銷活動)建議 [!DNL Google Ads] 每日不同時間的行銷活動層級競標修飾詞（廣告排程競標修飾詞），根據單一投資組合的每小時績效而定。<br><br>分析內容包括 [!DNL Excel] 具有個別工作表的活頁簿 [!UICONTROL Detailed Performance] （依行銷活動顯示的時間間隔的績效）與 [!UICONTROL Recommended TBAs] （每個行銷活動的建議廣告排程）。 您可以匯出 [!UICONTROL Recommended TBAs] 直接將工作表移至 [!DNL Google Ads] 編輯者。 分析也包含 [!UICONTROL Powerpoint] 含支援資料與說明的檔案。 |
